Search Help

The default search behavior is to search the "nmap_data" field, which is the raw output produced in the .nmap file. This can be overridden by prepending specific field names to searches, such as "ctime:" or "port_str:". For a complete list of field names, please see the json export of a scan. Currently, Natlas only supports searching the latest results for every host and does not offer the ability to search historical results.

  • The default operator is AND, but can be overridden using "OR"
    • 22 80 would search for results with both strings "22" AND "80" in them
    • 22 OR 80 would search for results with the string "22" OR "80" in them
  • Look for exact string matches by wrapping the term in quotes
    • "22/tcp" returns results in which the exact string "22/tcp" appears
  • Use "!" to exclude results
    • !ssh to filter out results containing the string ssh
  • Search for specific ports using "port_str:"
    • port_str:22 shows hosts with port 22 identified as open
  • Search for specific tags
    • tags:corpnet-a shows hosts tagged in the "corpnet-a" range
  • Search time ranges by using the "ctime:" keyword
    • ctime:<2019 to search for results before January 1st, 2019
    • ctime:2019-01-01 to search for results from January 1st, 2019
    • ctime:>2019-01 to search for results newer than January 1st, 2019
  • Put it all together now:
    • ctime:>2019-01 tags:corpnet-a port_str:22 !OpenSSH shows us results from corpnet-a that are newer than January 1st, 2019, listening on port 22, but do not contain the string "OpenSSH" in them
